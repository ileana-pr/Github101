# ✅ GitHub Security — Pre-Requisite Checklist

> Complete this **before** the session so we can hit the ground running!
> Estimated time: **10 minutes**

---

## Before This Workshop

### 1. 🐙 Completed the GitHub Introduction Workshop

This security workshop builds directly on the Introduction session. You should already have:

- [ ] A GitHub account with a verified email
- [ ] Git installed and configured (`git config --global user.name` and `user.email`)
- [ ] SSH key set up and connected to GitHub
- [ ] At least one repo you've pushed to

> [!IMPORTANT]
> If you haven't completed the Introduction workshop, start there first:
> [`introduction/student-resources/prereq_checklist.md`](../../introduction/student-resources/prereq_checklist.md)

---

### 2. 🔑 Know Which API Keys You're Using (Optional but Helpful)

If you're currently building something that uses API keys (OpenAI, Stripe, Supabase, etc.), have a note handy of:
- Which keys you have
- Which projects they're connected to

You don't need to share these — we'll use them as a reference when we set up your `.env` correctly.

---

### 3. 💻 Your Development Environment

Same as the Introduction workshop — you need **one** of:
- An AI IDE with a terminal (Cursor, Windsurf, PearAI)
- Any terminal on Mac/Linux/Windows (Git Bash)

---

## ✅ You're Ready If...

- [ ] You completed the GitHub Introduction workshop
- [ ] You have a terminal open and `git --version` returns a version number
- [ ] You can push to a GitHub repo from your machine
